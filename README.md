# Live Security Agent App

Machine learning-based methods to IoT botnet and threat detection have developed rapidly over the past couple of decades. In literature, the primary division is between the dominant paradigm of supervised classification and the opposing one of anomaly-based detection. However, both of these assumptions have their fragilities in IoT, and for different reasons.
Published detection models report near-perfect accuracy - but the majority of them are trained and tested on the *same* capture environment. Even though domain shift is acknowledged, it is rarely tested, which means a model can be confidently wrong in production, with no signal that it has left its competence. 

This is an area where Artificial Intelligence can compensate. If models trained on public IoT data degrade significantly when tested on independently captured traffic, can an LLM agent make the resulting degradation visible and interpretable?

## Methodology & Architecture
* supervised (Random Forest and XGBoost) and anomaly-based (Isolation Forest and Autoencoder) models trained on IoT-23 - Hide and Seek scenario
* homelab traffic generated by two Raspberry Pico 2W's acting as benign and respectively malicious IoT endpoints, with a Raspberry Pi 5 acting as a WAP and capture host
* LLM agent layer provided via Anthropic API

## Future Work
To further develop this concept, the following points should be pursued:
* added complexity to the testbed - real IoT devices, scale botnet simulation
* multi-scenario/multi-dataset training correlated to multi-tool agent development
* temporal/sequential feature interpretation by the agent
* local open-weight models to combat API costs and latency limitations
